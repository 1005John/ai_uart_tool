# 变更日志

## [v1.5.1] - 2026-06-04

### 修复
- 灵畿提交失败：只读模式关闭未检查返回值，readonly_off 增加自动重试和结果校验
- Windows 下 `lc bug create` 参数含换行符导致 `cmd.exe` 参数解析错位，`-D` 描述中的换行替换为空格
- webui.py 中引号冲突导致 SyntaxError

### 改进
- 灵畿提交成功后自动保存项目和责任人到 `~/.ai_uart_keys.json`
- 责任人文本框不再默认填充，改为用户自行填写，placeholder 提示可用人员
- 提交失败时显示具体错误信息（原为笼统提示）
- `-p`/`-w` 短选项改为 `--project-id`/`--workspace-key` 长选项，提高可读性
- 增加 `_run_lc` 调试日志（data/lc_debug.log）
- 缺陷详情描述改为可编辑 TextArea，支持修改后保存（`LocalDefectStore.update_description`）
- 缺陷详情支持粘帖/上传截图图片，自动保存到 `defects/attachments/`，Markdown 引用嵌入描述

## [v1.5.0] - 2026-06-04

### 新增
- 用例支持自定义期望值：用例集和方案管理的 Dataframe 增加"期望值"列，支持正则表达式匹配
- 方案执行时读取用户填写的期望值替代硬编码 `OK`，实现灵活的结果校验
- 缺陷管理"刷新缺陷"按钮：按筛选项刷新，无筛选时默认显示全部
- 执行完成后显示日志和缺陷文件路径（`data/last_exec_log.json` / `defects/`）

### 修复
- 方案 JSON 文件损坏导致崩溃：`load_plan()` / `load_test_set()` 加 `try/except` 保护
- 方案/用例集加载返回 `None` 时调用方空指针保护
- Windows 兼容：移除 `/usr/lib/python3/dist-packages` 硬编码（仅 Linux 时添加）
- 端口占用提示：杀掉旧进程后重新启动

### 文档
- README.md / CLAUDE.md 补充 Windows 安装和运行说明
- Windows 候选路径添加到 Excel 加载器

## [v1.3.0] - 2026-06-01

### 新增
- AT 口自动探测：扫描端口时自动发 AT+CGMM/AT+CGMR，下拉标签标注 `[ML307H | DU-MBRH3S00]`/`[DATA]`/`[无响应]`
- 多编码解码支持：`_safe_decode()` 工具函数，UTF-8 → GBK → Latin-1 自动回退
- 模组信息自动填充缺陷：连接串口时探测型号和固件版本，生成本地缺陷自动写入描述
- 状态框显示模组详细信息（型号、固件版本）

### 修复
- 串口数据解码乱码：二进制 PPP 口（ttyUSB1）不再误判为 AT 口，增加可打印 ASCII 比例检测(>70%)
- 扫描函数模组端口识别优化：PP 二进制数据含 `\r\nOK\r\n` 字节序列导致误判
- 测试集名尾部下划线：`list_test_sets()` 中 `[:-8]` → `[:-9]`，`_slug()` 增加 `rstrip('_')` 防护
- AT 指令截断：执行方案时 Dataframe 列从 `[:40]`/`[:45]`/`[:55]` 扩展至 `[:200]`
- webui.py 自动添加 user site-packages 路径，修复 Gradio 启动依赖缺失
